# 外部法規查詢結果 — 層疊卡片式頁籤 UI

> 線上展示：https://bryanhsiao.github.io/report_ui_design/

根據查詢條件動態產生頁籤，以層疊卡片的視覺呈現切換不同類別的法規查詢結果。

---

## 開發前規劃

### Context
建立一個報表查詢頁面，透過 URL 參數決定顯示哪些頁籤（法規資料、函釋資料、裁罰案件），點擊頁籤切換下方 jqGrid 表格內容。目前先用假資料開發。

### 技術選擇
- **純 HTML / CSS / JavaScript**（無框架）
- **jqGrid**（jQuery + jqGrid CDN）
- URL 參數格式：`?type=法規資料,函釋資料,裁罰案件`

### 檔案結構
```
index.html      — 主頁面（頁籤 + 卡片容器）
css/style.css   — 樣式（層疊卡片、動畫、Grid 覆寫）
js/app.js       — 主邏輯（URL 解析、頁籤切換、Grid 初始化）
js/mockData.js  — 假資料（三類報表共用欄位，不同資料）
```

### Grid 欄位定義
| 欄位名 | 中文名 | 說明 |
|--------|--------|------|
| lawName | 法規名稱 | 主要內容欄 |
| lastModified | 最後修正日期 | 日期 |
| authority | 主管機關 | 發布機關 |
| relatedRule | 相關聯內規 | 「查詢」連結或「-」 |

## UI 設計演進

### 1. 層疊卡片概念

每個頁籤對應一張卡片，多張卡片以右下偏移的方式堆疊，呈現出層次感：

```
┌──────────────────────────┐
│  前方卡片（active）        │
│                          │  ┐
│                          │  │ 第2張卡片邊框露出
└──────────────────────────┘  │
   └──────────────────────────┘
```

- 前方卡片完整顯示內容，後方卡片只露出彩色邊框
- 三種類別各有代表色：藍（法規）、青（函釋）、橘（裁罰）

### 2. 延遲載入 + Loading 過渡

- 初始只建立空卡片殼，**點擊頁籤才撈取資料**
- 切換時先顯示 spinner loading 畫面，避免白屏
- 已載入過的頁籤不重複載入，切回時直接顯示
- 模擬 800ms API 延遲（之後替換成真實 API）

### 3. 滿版自適應佈局

- 整個頁面佔滿 viewport（100vh），無多餘邊距
- 卡片高度隨視窗自動調整
- jqGrid 動態計算高度填滿卡片剩餘空間
- 內容不足時不出現捲軸，內容過多時由 Grid 內部捲動

### 4. 頁籤設計

- 白底乾淨風格，active 頁籤與卡片視覺連接
- 頂部色條標示類別，非選中頁籤以淺灰底區分

## 使用方式

直接用瀏覽器開啟 `index.html`，或透過靜態伺服器：

```bash
npx serve .
```

### URL 參數控制頁籤

| URL | 顯示的頁籤 |
|-----|-----------|
| `index.html` | 全部三個（預設） |
| `index.html?type=法規資料,裁罰案件` | 法規資料、裁罰案件 |
| `index.html?type=函釋資料` | 僅函釋資料 |

---

## 技術細節

- **卡片切換動畫**：CSS `transform` + `transition`（0.45s cubic-bezier）
- **Grid 寬高自適應**：載入後透過 `setGridWidth` / `setGridHeight` 動態設定
- **Flexbox 佈局**：header、tab-bar、卡片區以 flex column 分配空間
